\chapter{Web Services}

Die Java EE Web Services (JAX-WS) nutzen das SOAP-Protokoll um mit entfernten Diensten zu kommunizieren. Allerdings ist ein SOAP-Webservice oft ein Overkill und eine einfache RMI- oder REST-Schnittstelle tut es auch. Eine SOAP-Architektur besteht aus einem Client, einem Server und einem UDDI (Universal Description, Discovery and Integration). Der Server definiert seine Services mit der WSDL-Sprache (Web Service Description Language) und teilt diese Information dem UDDI mit.
